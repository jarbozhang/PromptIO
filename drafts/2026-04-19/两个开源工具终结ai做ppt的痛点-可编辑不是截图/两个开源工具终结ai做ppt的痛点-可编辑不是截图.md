# 两个开源工具终结AI做PPT的痛点，可编辑不是截图

上周帮朋友用某个热门 AI PPT 工具做路演 deck，生成出来确实好看。然后他说"帮我把第三页的数字改一下"，我点了半天才意识到，每一页都是一张图片。

改不了。

这个问题困扰我很久了。市面上绝大多数 AI 做 PPT 的方案，说到底是生成一张张截图然后塞进幻灯片。视觉效果没问题，但你想调个字号、换个颜色、删一段话？对不起，请重新生成。这不叫"做 PPT"，这叫"看 PPT"。

最近刷到两个开源项目，思路完全不同。我都跑了一遍，发现它们各有各的打法，但都在解决同一个问题，输出的东西你能真正编辑。

## html-ppt-skill，轻量到离谱

先说轻的这个。html-ppt-skill 是一个 Claude Code 的 skill，安装就一行命令。

```
npx skills add html-ppt-skill
```

装完之后你直接跟 Claude Code 说需求就行，比如"做一份 AI 分享，小红书风"。它会自动走完大纲、排版、选主题、加动效这一整套流程。

让我比较意外的是它内置了 36 套主题和 31 种布局模板。坦率讲，对于大多数场景来说，你根本不需要自己设计，选一套主题说一句话，出来的东西就能用。它甚至支持粒子特效和动画，展示效果接近视频级别。

不过有个地方需要说清楚，它的导出格式是图片，适合直接发图文内容。如果你的需求是"做完发小红书"或者"做完发朋友圈"，这个工具的效率很高。但如果你要的是一个能在 PowerPoint 里逐元素编辑的 .pptx 文件，它不是为这个场景设计的。

所以我把它定位成"轻量级内容生成器"。850 个赞不是没道理的，它确实把"一句话出 PPT"这件事做到了足够丝滑。

但我真正想聊的是下面这个。

## PPT Master，真正的可编辑 PowerPoint

PPT Master 做的事情不一样。

它输出的每个元素都是真实的 PowerPoint 形状。你可以点击，可以拖动，可以编辑文字，可以直接发给老板。不是截图，不是图片，是原生 .pptx。

5780 星，开源，免费。

这个项目背后的故事也值得说一下。作者 Hugo He 是金融从业者，做投行咨询的，每天审阅几百张 PPT。他受不了市面上那些"导出图片"的方案，干脆自己写了一个。说真的，最好的工具往往来自最痛的人。

用法也不复杂。

```
git clone <仓库地址>
pip install -r requirements.txt
```

把你的 PDF、DOCX 或 Markdown 文件丢进 projects/ 目录，告诉 Claude Code"请用这个文件生成 PPT"，等它跑完，去 exports/ 目录拿文件。

整个处理在本地跑，文件不用上传到任何服务器。对于金融、法律这些对数据敏感的行业来说，这一点很重要。

我认为 PPT Master 解决的其实是一个被忽视了很久的问题。大家都在卷"AI 生成 PPT 有多好看"，但几乎没人认真想过"生成出来的东西能不能改"。可能有些想法还不成熟，但我觉得"可编辑"这个属性才是 AI PPT 工具的及格线，而不是加分项。

你想想看，现实场景里有几个 PPT 是一次生成就完美的？老板要改数字，客户要换 logo，同事要加一页。如果每次修改都要重新生成整套，那 AI 帮你省的时间又全还回去了。

## 两个工具怎么选

反正我觉得逻辑很简单。

如果你的终点是社交媒体图文，html-ppt-skill 更合适，一句话出图，够快够好看。

如果你的终点是一份要反复修改、最终发给甲方或老板的 .pptx 文件，PPT Master 是目前我见过的最靠谱的开源方案。

两个都配合 Claude Code 使用，学习成本很低。

坦率讲，AI 做 PPT 这个赛道已经太拥挤了，但大部分产品都在比谁的模板更花哨。真正的痛点从来不是"好不好看"，而是"改不改得动"。这两个开源项目，一个轻一个重，但都站在了正确的那一边。

下次再有人跟你安利"AI 一键做 PPT"，你可以多问一句，生成出来的，能编辑吗？
