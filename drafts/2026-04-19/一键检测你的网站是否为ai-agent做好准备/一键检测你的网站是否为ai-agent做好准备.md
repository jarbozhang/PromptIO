# 一键检测你的网站是否为AI Agent做好准备

我刚在 Hacker News 上看到一个工具，输入你的网站 URL，它帮你扫描五个维度，告诉你你的网站对 AI Agent 友不友好。108 个赞，174 条评论，吵翻了。

我立刻跑去测了几个网站。结果挺有意思的，但更有意思的是评论区暴露出来的一个根本矛盾。

## 这工具到底查什么

isitagentready.com，Cloudflare 出品。用法极简，输入 URL，点扫描，几秒钟出结果。

它查五个大类。

**可发现性**，你的 robots.txt 是否存在、Sitemap 是否完整、Link response headers 是否正确。这些是基本功，跟传统 SEO 没什么区别。

**内容可访问性**，你的页面能不能以 Markdown 格式被读取。Agent 不喜欢解析 HTML 里一堆广告和导航栏，它们更想拿到干净的文本。

**机器人访问控制**，你的 robots.txt 里有没有针对 AI 爬虫的专门规则。注意，不是屏蔽它们，是告诉它们哪些能抓、哪些不能。

**协议发现**，你的网站有没有暴露 MCP 服务器、Agent Skills、OAuth 端点这些新玩意。这部分是最前沿的，大部分网站目前都不会有。

**商务能力**，你的网站能不能让 Agent 直接付款交易。支持 x402、UCP、ACP 这些协议。说实话，这块目前更像是一个愿景，而不是现实需求。

我测了几个网站，大部分得分在 0 到 25 分之间。这不意外，因为除了 robots.txt 和 Sitemap，后面那几项根本就是新标准，几乎没人实现过。

## 评论区吵起来了

HN 评论区最热的一条评论，直接戳到了要害。

大意是说，AI 行业一边声称自己能替代白领工作，一边又反过来要求全世界的网站为它们做适配。这不是很矛盾吗？如果你的 Agent 真那么聪明，它应该能处理任何网页，而不是要求网站主动喂数据。

这个观点获得了大量共鸣。

还有一个尖锐的声音来自内容创作者。有人说自己网站的流量因为 AI 摘要功能下降了 60%，现在又被要求帮助这些 AI 平台更好地抓取自己的内容。坦率讲，这种愤怒完全可以理解。

但也有人提出了不同的角度。

有评论者把这比作早期的 SEO。没人喜欢为 Google 优化网站，但如果 Google 是你流量的主要来源，你就得做。同样的逻辑，如果 AI Agent 将来成为一个重要的流量渠道，网站就不得不适配。关键问题是，AI 流量到底能不能转化成实际的商业价值。

一个做本地餐厅网站的开发者说得挺实在的，他不在乎什么意识形态争论，他只关心顾客能不能通过 Agent 找到他的餐厅、看到菜单、完成预订。如果"Agent Ready"能帮到这一点，他愿意做。

## 讽刺的一幕

有人试着用这个工具扫描 isitagentready.com 自己。

返回了 522 错误。

这个工具没法检测它自己是否 Agent Ready。评论区直接炸了。虽然这可能只是个 Cloudflare 边缘节点的临时问题，但你不得不承认，这个画面实在太有喜剧效果了。

还有人拿 motherfuckingwebsite.com 去测，那个极简到只有纯文本的网站，得了 0 分。但那个网站恰恰是最容易被任何 Agent 读取的页面。这就暴露了评分标准的一个盲点，它衡量的是你是否实现了特定的新协议，而不是你的内容是否真的对 Agent 可用。

## 我的判断

我认为这个工具本身的价值有限，但它揭示的趋势值得认真对待。

先说工具本身。五个检测维度里，前两个（可发现性和内容可访问性）是有实际意义的，做好 robots.txt、Sitemap、提供干净的文本内容，这对搜索引擎和 AI Agent 都有好处。后面三个维度目前更像是 Cloudflare 在推自己的生态标准，实用性还没被验证。

再说趋势。"Agent Ready"这个概念正在从一个 buzzword 变成一个真实的需求。想想看，当越来越多的用户通过 Claude、ChatGPT 这些工具的联网功能来获取信息，你的网站如果完全不考虑这些 Agent 的访问体验，你可能会错过一个正在增长的流量入口。

但我也不认为需要现在就急着去实现 MCP 服务器或者 x402 支付协议。这些太早了。

我自己也还在摸索，但目前我觉得一个务实的优先级是这样的，先把基础做好，一个规范的 robots.txt，一个完整的 Sitemap，页面内容尽量语义化。如果你有余力，考虑加一个 llms.txt 文件，告诉 AI Agent 你的网站是什么、有哪些核心页面。这个文件规范很简单，成本几乎为零。

至于那些花哨的协议和标准？等它们真正被主流 Agent 采用了再说。现在入场，大概率是在给 Cloudflare 的生态做免费测试。

## 你该做什么

打开 isitagentready.com，输入你的网站 URL，跑一次。

不用太在意总分，重点看可发现性那一栏。如果你的 robots.txt 和 Sitemap 都没问题，你已经比大多数网站做得好了。如果有红叉，花 10 分钟修一下。

然后问自己一个问题，你的网站内容，在去掉所有导航栏、侧边栏、广告之后，是否还能被清晰地理解？如果能，你的网站对 Agent 就已经足够友好了。

SEO 用了十年才从一个可选项变成一个必选项。Agent Ready 可能也在走同一条路，只是这次，速度会快得多。
