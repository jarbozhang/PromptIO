# 微信支付官方为AI Agent开源Skills

上周我在 Cursor 里写一个小程序支付功能，签名又报错了。第三次了。我盯着文档里那堆 HMAC-SHA256 的参数顺序，突然想，2026 年了，接微信支付怎么还是这么痛苦？

然后我刷到了一条消息，微信支付官方在 GitHub 开源了一套 AI Agent Skills。

我第一反应是，又一个 demo 级别的东西吧？点进去一看，不是。这东西是认真的。

## 它解决的是一个真实的痛点

接过微信支付的人都懂，最难的不是调 API，是选型和签名。

JSAPI、APP、H5、Native、小程序、付款码、合单支付，光支付方式就七八种。每种的签名方式、回调格式、参数要求都不一样。你去问 Claude 或者 ChatGPT，它们会很自信地给你一段代码，签名逻辑是编的，回调地址格式是错的，参数名看着像但就是不对。

为什么？因为大模型的训练数据里，微信支付的文档版本混杂，v2 和 v3 的接口混在一起，社区里的过时代码到处都是。AI 越自信，你踩的坑越深。

微信支付这套 Skills 干的事情，就是把 AI 的"自由发挥"关进笼子里。

## 三层结构，每层都有用

我拉了 GitHub 仓库（wechatpay-apiv3/wechatpay-skills，MIT 协议，目前 183 个 star），仔细看了一遍结构。

每个 Skill 对应一个微信支付产品，目前开源了两个，基础支付（wechatpay-basic-payment）和商家券（wechatpay-product-coupon）。每个 Skill 包含三层内容。

第一层是业务知识。接入流程、参数说明、状态流转，全部结构化整理好，Agent 可以直接检索。不是那种"请参考官方文档"的敷衍，是把文档拆成了 Agent 能理解的知识块。

第二层是代码示例。覆盖各接口的 API 调用示例，含签名。语言覆盖 Java、Go、Python（仓库语言占比 Java 52.5%、Go 45.7%、Python 1.8%）。关键点在于，这些示例是官方维护的，签名逻辑是对的。

第三层是交互规范。这层最有意思，它定义了 Agent 的能力边界。比如 SKILL.md 里明确写了，"只检索不生成"，代码示例必须从文档检索，不能让 AI 自己编。"只展示不写入"，禁用 write_to_file 等文件修改工具。还有"先交互后输出"，提供代码前必须确认商户模式、开发语言、具体接口。

这第三层规范，坦率讲，是我觉得最值得学习的设计。它不是在增强 AI 的能力，而是在约束 AI 的行为。

## 五个场景，覆盖了支付接入的全流程

具体能干什么？五件事。

产品选型，你告诉 Agent 你的业务场景，它帮你判断该用 JSAPI 还是 APP 还是 H5，包括是否需要合单支付。

示例代码，下单、调起支付、回调处理、退款、对账，每个接口都有可直接参考的代码。

知识问答，订单状态怎么流转、退款规则是什么、APPID 怎么绑定，随问随答。

质量检测，上线前帮你扫一遍代码，检查签名验签、回调规范、必接接口有没有漏。

排障，报错了把错误码丢给它，直接定位原因。

说真的，前四个加在一起，基本就是一个资深微信支付开发者坐你旁边 pair programming 的体验。

## 这件事为什么重要

你可能觉得，不就是把文档塞给 AI 吗？有什么了不起的？

区别在于谁在做这件事。

社区里早就有人把微信支付文档丢进 RAG 系统了，但效果参差不齐。文档版本不对、知识粒度太粗、没有交互规范约束 AI 行为，这些问题个人开发者很难解决。

微信支付官方亲自下场，意味着几件事，知识是权威的、代码是验证过的、交互规范是经过内部测试的。而且用 MIT 协议开源，你可以直接在 Cursor、Claude Code 或者任何支持 Skills 的 AI IDE 里用。

我认为这代表了一个趋势，API 提供商会越来越多地为 AI Agent 提供官方的"技能包"，而不是让开发者自己去 RAG。

反过来想，如果你是一个 API 提供商，你的开发者正在用 AI 写接入代码，但 AI 写出来的代码全是错的，这对你的品牌伤害有多大？微信支付是第一个想明白这件事的大厂支付团队。

## 我的判断

这套 Skills 目前还处于早期。只开源了基础支付和商家券两个产品，语言覆盖也不算全面（PHP 和 .NET 开发者暂时只能看着）。

但方向是对的。

我自己也还在摸索怎么把它集成到日常开发流程里，但有一点已经很确定了，以后写微信支付相关的代码，我不会再让 AI 自由发挥了。官方给了笼子，就该用笼子。

可能有些想法还不成熟，但我甚至觉得，半年后各大支付平台（Stripe、支付宝）都会跟进类似的方案。谁先做，谁就在 AI 开发者生态里占到先机。

去试试吧。仓库地址在下面，clone 下来丢进你的 AI IDE，下次接微信支付的时候，你会感谢自己的。
