# OpenAI充值漏洞，0元激活200刀Pro，还没修

有人 0 元激活了 200 美元的 ChatGPT Pro 订阅。

不是 Bug Bounty 测试，不是内部员工福利。是一个普通用户发现了 OpenAI 充值系统的验证漏洞，发到了推特上，已经有人照着成功跑通了。

截至我写这篇文章的时候，这个漏洞还没修。

## 到底是什么漏洞

核心问题四个字，验票不验人。

OpenAI 的订阅激活流程有一个环节，系统会根据一个充值收据来确认你的付款状态。但它只检查收据本身是否有效，不检查这张收据是不是你的。

打个比方，就像电影院检票，验了票是真票就放你进去，不管这张票是不是你买的。一张收据可以满天飞。

发现这个��洞的人叫 xkajon，他之前在推特上发了第一版帖子，被人一字不差地搬到了 OpenAI 官方社区。然后他的私信箱就炸了，好几个人发截图说照着操作成功激活了。

Plus 能搞，Pro 5x 能搞，Pro 20x 也能搞。

## 外面的代充市场

如果你不太关注这个灰色地带，可能不知道 ChatGPT 代充已经是一个成熟的产业了。

OpenAI 不支持中国信用卡直接订阅，所以国内用户要么用虚拟信用卡（Depay、Nobepay 之类），要么找代充。代充的市场价大概七八十人民币一个号。

还有一批"公益站"，打着做公益的旗号提供低价开通服务，但等待时间长。

这个漏洞如果被大规模利用，等于直接把代充市场归零。免费的东西谁还付钱。

xkajon 说他正在跟几个成功跑通的人做一个自动激活链接，准备免费放出来。

## 这说明什么

坦率讲，我关注这件事不是因为可以薅羊毛，而是因为这暴露了 OpenAI 的工程质量问题。

这是一家估值几千亿美元的公司。它的充值验证系统连"收据是不是你的"都不检查。

有人在评论区说"OpenAI 这么大的公司不可能这么草台班子"。结果还真就是。

如果你仔细看 OpenAI 过去半年的安全记录，会发现这不是孤例。Axios 供应链投毒事件波及了 macOS 应用，Codex 的权限控制被多次质疑，API Key 泄露导致天价账单的案例也不是第一次了。

快速扩张的代价。产品迭代速度太快，安全和支付这种"不性感但关键"的基础设施跟不上。

## 社区的态度

推特上 787 个赞，118 条评论。态度很分裂。

一派觉得"有便宜不占是傻子"，已经开始动手了。另一派担心法律风险，毕竟利用支付漏洞获取服务在很多国家属于违法行为。

我自己也还在摸索怎么看这件事。从技术上说，这确实是 OpenAI 的锅。但从道德上说，你知道这是漏洞还去用，跟捡到别人掉的钱包不还其实没什么区别。

还有人指出了一个更实际的问题，如�� OpenAI 修复后追溯封号怎么办？你用漏洞激活的账号里可能有大量对话记录和自定义 GPT，说封就封。

## 我的判断

我认为这件事有两层值得关注。

第一层是短期的，这个漏洞大概率很快会修。发到推特上、被搬到官方社区、还有人要做自动化工具，OpenAI 不可能不注意到。如果你真的想利用这个漏洞，窗口期可能只有几天。但我不建议你这么做，风险远大于收益。

第二层是长期的。一家想要主导 AI 时代的公司，支付系统的安全做成这样，这是一个信号。说明 OpenAI 的工程资源分配存在严重的偏科，模型能力拉满，基础设施安全不及格。

如果你是 OpenAI 的企业客户，看到这个消息应该会紧张。你的 API 调用数据、支付信息、甚至模型微调的数据，是不是也存在类似的验证缺陷？

这可能比免费薅 200 美元更值得思考。
