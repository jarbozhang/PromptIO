# 22岁创业者扒开Claude Mythos黑箱，全开源了

Anthropic 花了多少钱训练 Claude Mythos，外界不知道。但一个 22 岁的创业者带着几个人，只用公开论文和第一性原理，就把这个闭源模型的架构复现了出来。

然后他把全部代码扔到了 GitHub 上。

## 他发现了什么

这个项目叫 OpenMythos。核心猜测是 Claude Mythos 没有靠堆参数，而是用了一种叫循环深度 Transformer 的架构。

什么意思呢。

传统大模型像盖高楼，一层叠一层，参数越多楼越高。Claude Mythos 的思路不一样，它像原地跑步，同一套权重在一次前向传播里循环跑 16 次。

每循环一次，相当于多想了一步。所有思考都在隐藏空间里进行，不需要像思维链那样每步都吐出可见的 token。

这带来一个直接后果，770 亿参数的循环模型，能打平 1300 亿参数的普通模型。

对消费级硬件来说，这是天大的好消息。你用更小的显存就能跑出更强的效果。

## 为什么这件事重要

过去两年 AI 行业的 Scaling Law 是一条直线，模型越大越强，算力越多越好。

OpenMythos 指向了另一条路，不靠堆参数，靠增加思考深度。用时间换空间。

你想想，如果这条路走得通会怎样。NVIDIA 不再是唯一的赢家。小团队用消费级 GPU 也能跑出有竞争力的模型。AI 的成本曲线可能出现一个拐点。

当然，这些都是"如果"。OpenMythos 目前还是一个复现尝试，不是经过验证的生产级模型。但方向本身非常值得关注。

## 架构壁垒还能撑多久

推特上 1600 多个赞，评论区分成了两派。

有人觉得这是范式转移的信号，闭源实验室的架构优势会消失得越来越快。只要你发布了一个足够强的模型，总有人能通过逆向工程猜出你的架构。Anthropic 藏得再严，也挡不住公开论文和第一性原理。

但反过来想，架构只是模型能力的一部分，训练数据和对齐方法同样重要。你复现了架构，不代表你能复现效果。

我自己也还在摸索对这件事的判断。但有一点是确定的，22 岁能做到这种水平的逆向工程，说明 AI 领域的人才密度已经高到了一个惊人的程度。

## 技术细节

循环深度 Transformer 的核心思想并不新。学术界几年前就有人提了 Universal Transformer，思路类似。

但 OpenMythos 团队做的事情是把这个理论框架落地到了一个具体的、可运行的实现。

几个关键设计决策。

共享权重，16 次循环用的是同一套参数。这大幅减少了模型的存储需求。

隐空间推理，不像 Chain-of-Thought 那样把中间步骤都输出为文本，而是在 embedding 空间里直接迭代。噪声更少，组合能力更强。

自适应退出，不是所有输入都需要跑满 16 次。简单问题可能跑 4 次就收敛了。这进一步降低了推理成本。

## 我的判断

我认为 OpenMythos 的技术价值是真的，但短期内不会改变格局。

原因很简单，架构创新和工程落地之间还有巨大的鸿沟。Anthropic 的护城河不只是架构，还有数万亿 token 的训练数据、精心设计的对齐流程、以及调了无数遍的超参数。

但长期来看，这件事传递了一个清晰的信号，闭源的架构壁垒比大家想象的脆弱。真正的壁垒在数据和对齐，不在模型结构。

未来最强的模型不会是参数最多的，会是想得最深的。

去 GitHub 搜 OpenMythos，几行代码就能跑起来。
