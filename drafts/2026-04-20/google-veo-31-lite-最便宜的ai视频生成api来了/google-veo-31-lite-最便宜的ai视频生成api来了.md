# Google Veo 3.1 Lite，最便宜的AI视频生成API来了

AI 视频生成的问题从来不是"能不能做"，而是"做得起吗"。

Runway 一分钟视频几美元起步。Sora 到现在还没开放 API。Pika 的定价对个人开发者也不算友好。

然后 Google 把 Veo 3.1 Lite 扔了出来。定位很明确，它就是冲着"最便宜"来的。

## Veo 3.1 Lite 是什么

Google 的视频生成模型家族叫 Veo。3.1 是最新一代，Lite 是这一代里专门为成本优化的版本。

你可以理解为，Veo 3.1 是旗舰版，效果最好但价格贵。Veo 3.1 Lite 是性价比版，效果打个八折但价格可能只有十分之一。

Google 官方博客的原话是"most cost-effective video generation model"。最具成本效益。

它通过 Vertex AI 和 Gemini API 提供服务。意味着你可以用代码直接调用，不需要打开网页一个个生成。对开发者来说这很关键，因为你可以把视频生成嵌入到自己的产品流程里。

## 能做什么

从 Google 的演示来看，Veo 3.1 Lite 支持文本到视频生成和图片到视频生成两种模式。

文本到视频，你写一段描述，它生成几秒钟的视频片段。

图片到视频，你给一张静态图，它让图里的东西动起来。这个用法在电商场景特别有价值。你有一张产品静态图，让模型给它加上旋转、光影变化、使用场景演示。

视频时长和分辨率的具体参数，Google 暂时没有完全公开。但从 Veo 系列之前的版本来看，应该支持至少 720p、几秒到十几秒的片段。

## 谁会用这个

我想了三个场景。

第一，电商和内容创业者。你有一个淘宝店，需要给每个产品做一段短视频展示。以前找人拍要几百块一条，现在 API 调一下可能几毛钱。一天做一百条也不心疼。

第二，社交媒体自动化。你做短视频账号，需要大量素材。文本转视频 API 可以让你批量生成背景画面、过渡动画、封面动图。

第三，教育和培训。做课件的时候需要演示动画，以前用 PPT 动画凑合，现在可以生成更生动的视频片段。

## 跟竞品比呢

Runway Gen-3 Alpha 效果目前是天花板级别，但价格也是天花板。适合专业创作者和有预算的团队。

Sora 还没开放 API。只能通过 ChatGPT 界面用，没法集成到自己的产品里。

Pika 和 Kling 在国内有一定用户基础，但 API 的文档和稳定性跟 Google 比还有差距。

Veo 3.1 Lite 的定位不是跟 Runway 比效果，而是跟所有人比价格。Google 有一个其他公司都没有的优势，它的 GPU 集群是自己建的。别人要去 NVIDIA 买卡，Google 自己有 TPU。硬件成本低，模型推理就能便宜。

反正我觉得，如果你不是做专业影视后期，Veo 3.1 Lite 的效果应该够用。八分效果、一折价格。

## 我的判断

我认为 Veo 3.1 Lite 不会改变视频创作的上限，但会大幅提高下限。

以前做视频内容有一个隐性门槛，你得有拍摄设备、有剪辑能力、有素材库。现在一个 API 调用解决了"素材从哪来"的问题。

对中国开发者来说，有一个问题需要注意，Vertex AI 的服务在国内访问可能有延迟。如果你是做面向国内用户的产品，可能需要考虑用国内的替代方案，或者在海外服务器上做一层代理。

不过作为开发测试和小规模使用，直接调 API 完全可行。

建议先去 Google AI Blog 看看具体的定价和 API 文档。如果你手头有一个需要视频素材的项目，现在是最好的试水时机。
