# 7个AI玄学Skills，赛博算命/看风水/批八字，古籍知识库治好AI幻觉

我上周让 Claude 帮我排了个八字。它一本正经地告诉我"你的命格属于从革格"，还引用了《滴天髓》。

问题是，它引的那段原文根本不存在。

这就是大模型做玄学最尴尬的地方，它什么都敢说，但你完全没法验证。古籍那套专有术语体系，对 LLM 来说就是幻觉高发区。模型训练语料里《穷通宝典》可能就几个片段，剩下的全靠"创意补全"。

但最近冒出来一批 Claude Code Skills，用了个简单粗暴的办法治这个病，直接把古籍原文做成知识库塞进去。

我花了一个下午挨个试了 7 个，有些确实让我愣了一下。

## 最硬核的，八字命理 Skill

先说让我印象最深的。jinchenma94/bazi-skill 这个项目，你在 Claude Code 里输入"算八字"就能触发。它会一步步收集你的出生信息，阳历阴历都支持，然后自动排出四柱。

关键在后面。它内置了九本经典古籍作为参考源，包括《穷通宝典》《三命通会》《滴天髓》。分析日主强弱、十神关系、五行平衡的时候，推理链条是能对上这些古籍的核心方法论的。

我拿自己的八字测了一轮，追问了三四个问题。和之前裸跑 Claude 的最大区别是，多轮追问下来逻辑没有散。之前那种"前面说你五行缺水，后面又说你水旺"的自相矛盾，在这个 Skill 里确实没遇到。

坦率讲，安装方式也足够简单，把文件放到 `.claude/skills/bazi/` 目录下就行。

## 一个人搞了十二套系统的疯子

然后是 hhszzzz/taibu，这个项目让我有点震撼。

一个人把八字、紫微斗数、奇门遁甲、六爻、塔罗、梅花易数、太乙、大六壬、星座、MBTI、面相、解梦，十二套系统全做进了一个平台。

技术上它做了几件事让我觉得不是玩票的。第一，八字排盘用的是真太阳时，不是简单拿北京时间算。搞过命理的人知道，时辰差一个，整个盘都不一样。第二，它标记了 51 个神煞，这个覆盖面相当细了。第三，它实现了 MCP Server，意味着其他应用可以直接调用它的排盘能力。

更有意思的是它的混合授权模式，核心计算库用 MIT 开源，Web 端用 AGPL。坦率讲...不对，其实就是，算法你随便用，但你想直接抄我的网站界面，那得谈。

这个项目支持 Docker 一键部署，`docker compose up -d --build` 就能跑起来。

## 佛学 Skill 的"硬门禁"设计

xr843/Master-skill 走了一条完全不同的路。它做了八个历史上真实存在的高僧人设，玄奘讲唯识，慧能讲禅宗，智顗讲天台。

但让我真正停下来想了一下的是它的防幻觉机制。

它接入了 FoJin.app 的 503 个数据源，67.8 万条语义向量，3.1 万个实体知识图谱。所有引用必须带 CBETA 典藏编号，格式是 Txxn####。你问一个问题，它如果在典藏里找不到对应的出处，就不会编一个给你。

这个设计叫"HARD-GATE"。没有 CBETA 引用，不准输出教义内容。没有原典出处，不准虚构。

我认为这是目前我见过的、在垂直领域对抗 LLM 幻觉最优雅的方案之一。不是靠 prompt 里写"请不要编造"，而是从架构层面卡死，没有源就没有输出。

它甚至做了 fidelity 测试集，每个法师人设都有 5 组以上的 Q&A 验证用例，跑 CI 流水线。

说真的，一个玄学 Skill 比很多正经 RAG 应用的质量保障都到位。

## 剩下四个，快速过一遍

**奇门遁甲 + 紫微斗数**（FANzR-arch/Numerologist_skills），把奇门的用神格局和紫微的安星四化做了工程化拆解。适合对这两个体系有基础认知的人。

**月老姻缘测算**（Ming-H/yinyuan-skills），专攻感情方向，把八字合婚、生肖配对、求签整合在一起。模块化设计，想测哪个测哪个。

**易经风水阳宅分析**（voidforall/fengshui.skill），输入住宅方位布局就能跑分析。我拿自己工位试了一下，它建议我把显示器往左挪 15 度。你品品。

**塔罗牌解读**（daman-ovo-0404/tarot-skill），支持随机抽牌和多种牌阵。日常纠结选 A 还是选 B 的时候，当个多视角灵感触发器挺有意思。

## 我的判断

这波玄学 Skills 背后其实藏着一个严肃的技术命题，垂直领域的知识密度越高、术语体系越封闭，LLM 裸跑的幻觉率就越离谱。

我认为"古籍知识库 + 强制引用门禁"这个模式，完全可以复制到中医、法律、金融合规这些同样依赖专业文献的领域。Master-skill 的 HARD-GATE 设计不是玄学的专利，它是所有高置信度 RAG 应用都应该学的范式。

可能有些想法还不成熟，但我觉得 Skills 生态正在成为 Claude Code 最被低估的扩展点。这 7 个项目没有一个需要你写后端、搭服务器、申请 API key。放进目录，输入关键词，就能跑。

反正我觉得，命运这事儿信不信都无所谓。但这些项目解决"AI 不要瞎编"的思路，是真的值得偷。
