# VoxCPM 2冲上GitHub Trending，中国开源TTS做到30种语言可商用

一个 TTS 模型，不需要你录一句话，只靠打字描述"30岁女性、温柔、带点慵懒"，它就能凭空造出一个声音来。

我第一反应是不信。

VoxCPM 2 这两天冲上了 GitHub Trending，来自清华 OpenBMB 团队。2B 参数，30 种语言，48kHz 音质，Apache 2.0 可商用。我花了点时间把仓库翻了一遍，发现这东西确实有几个地方让我停下来想了想。

## 纯文本造声音，这事为什么重要

做过语音产品的人都知道，传统 TTS 的痛点不在合成质量，而在声音资产。你想要一个"年轻男性、略带沙哑、说话偏快"的声音？要么去录音棚找人录，要么从有限的预设里凑合选一个。

VoxCPM 2 的 Voice Design 功能直接绕过了这一步。你用自然语言描述你想要的声音特征，它就生成。不需要参考音频，不需要预设声库。

坦率讲，这不是第一个做 Voice Design 的模型。但把它和 30 语言支持、48kHz 输出、可商用许可打包到一个 2B 参数的统一模型里，这个组合确实少见。

## 三种玩法，层层递进

VoxCPM 2 的能力分三档，越往后越"变态"。

第一档是基础合成。给文本，出语音，30 种语言自动识别，不需要手动指定语言标签。支持中文、英文、日语、韩语这些常见的，也覆盖了阿拉伯语、斯瓦希里语、老挝语这些小语种。甚至四川话、粤语、吴语、东北话这些中文方言也在列。

第二档是可控克隆。给一段短音频，它能克隆这个声音，同时你还能叠加风格控制，比如让克隆出来的声音"更兴奋一点"或者"放慢语速"。

第三档是它叫 Ultimate Cloning 的东西。你给参考音频加上对应的文字稿，它生成的语音会忠实还原说话人的每一个细节，包括呼吸节奏、语气词、口癖。说真的，听 demo 的时候我反复确认了好几次，那种停顿和气息感不像是合成出来的。

## 跑起来需要什么

这可能是最让我意外的部分。

8GB 显存就能跑。

安装就一行 `pip install voxcpm`，Python 代码也很直白，五行就能出声。在 RTX 4090 上实时率 (RTF) 大概 0.3，意思是生成 1 秒语音只需要 0.3 秒。如果用他们的 Nano-vLLM 加速方案，能压到 0.13。

对于独立开发者来说，这个门槛已经低到可以直接塞进产品原型了。

不过我也得说，2B 参数模型在消费级显卡上跑 30 种语言，效果不可能每种都完美。我的判断是，中英日韩这几个主力语种的效果应该不错，小语种大概率还有打磨空间。毕竟 200 万小时的训练数据分摊到 30 种语言，每种语言分到的量差异肯定很大。

## 我认为这件事的意义不只是又一个 TTS

开源 TTS 这条赛道过去一年挤满了选手。从 ChatTTS 到 Fish Speech 到 Qwen3-TTS，每隔几个月就有一个新模型出来。

但 VoxCPM 2 做对了一件事，它把"声音设计"这个能力从专业工具里解放出来了。

你想想看，以前做一个多语言有声读物 App，光声音资产这一项就能卡住小团队半年。现在？用文本描述生成角色声音，用短音频克隆真人声音，用风格控制调整情绪。整条链路都在一个模型里闭环了。

我认为 VoxCPM 2 对独立开发者和小团队的价值，远大于它对大厂的价值。大厂有自己的语音团队和数据，但小团队需要的就是这种"开箱即用、Apache 2.0 可商用"的方案。

可能有些想法还不成熟，但我觉得 TTS 赛道正在经历和图像生成类似的路径，从"大厂专属"快速滑向"人人可用"。VoxCPM 2 不是终点，但它把门槛又往下踹了一脚。

## 想试的话

如果你手头有一张 8GB 以上显存的显卡，今天就可以跑起来。

如果没有本地算力，Hugging Face Spaces 上有在线 demo 可以先玩玩。

与其在这篇文章下面点收藏，不如打开终端敲一行 `pip install voxcpm`，三分钟之后你就知道这东西适不适合你的场景了。
