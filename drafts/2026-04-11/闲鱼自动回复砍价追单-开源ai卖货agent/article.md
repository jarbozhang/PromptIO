我在闲鱼上挂了十几件闲置，最头疼的不是没人问，是问的人太多了。

晚上十一点有人砍价，早上七点有人追问发货时间，中午又有人问"包邮吗"。你要是秒回，成交率能高一截。你要是隔了俩小时才看到消息，人家早去别家了。

但我不可能 7x24 小时盯着手机。

前两天刷到一个开源项目 XianyuAutoAgent，说是专门给闲鱼卖家做的 AI 自动值守工具。6800 个 Star，1200 个 Fork，GPL-3.0 协议。我花了一个小时把它跑起来了，说说真实感受。

## 它到底能干什么

三件事，对应闲鱼卖家最痛的三个场景。

**自动回复**。买家问问题，LLM 根据你商品的描述自动生成回答。不是那种关键词匹配的死板机器人，它会读上下文，知道买家之前问了什么。你可以在 prompts 目录下自定义提示词，分成意图分类、价格谈判、技术咨询、默认回复四个文件，各管各的。

**智能砍价**。这个是我最感兴趣的部分。它用 LLM 加规则引擎做阶梯降价，不是一刀切地"最低 XX 元"，而是根据砍价轮次逐步让步。该硬的时候硬，该松的时候松。你在提示词里设定底价和策略，剩下的交给模型判断。

**自动追单**。买家聊了半天没下单，它会自动跟进。这个功能还在早期，但逻辑是对的，闲鱼上太多"聊完就忘"的潜在买家了。

## 跑起来的过程

坦率讲，安装过程比我预想的简单。

```
git clone https://github.com/shaxiu/XianyuAutoAgent.git
pip install -r requirements.txt
```

核心配置在 `.env` 文件里，你需要填四样东西。

`API_KEY`，你的 LLM API 密钥。`COOKIES_STR`，闲鱼网页端的 Cookie，F12 打开 Network 面板抓。`MODEL_BASE_URL`，模型服务地址。`MODEL_NAME`，模型名称，默认用的通义千问。

Cookie 这一步是最麻烦的。你得登录闲鱼网页版，打开开发者工具，从请求头里把 Cookie 复制出来。每隔一段时间 Cookie 会过期，需要重新抓。

配好之后 `python main.py` 就跑起来了。

我用通义千问测了一下，回复速度大概 3-5 秒，对于闲鱼场景完全够用。模型支持换，只要兼容 OpenAI API 格式的都行，你想用 DeepSeek、GPT-4o 也可以。

## 我觉得有意思的设计

项目结构很清晰，核心就五个文件。`XianyuAgent.py` 是 Agent 主逻辑，`XianyuApis.py` 封装闲鱼接口，`context_manager.py` 管上下文，`main.py` 是入口。提示词全放在 prompts 目录下，改起来很方便。

它的意图分类设计值得说一下。买家发消息进来，先过一个分类器判断意图，是砍价、技术问题还是普通咨询，然后路由到不同的"专家"提示词去处理。这个思路其实就是现在 Agent 架构里常见的 Router 模式，只不过用在了电商客服场景。

还有个细节，它支持模拟人工打字延迟。开启 `SIMULATE_HUMAN_TYPING` 之后，回复不会瞬间蹦出来，而是模拟真人打字的速度慢慢出。这个对防止被平台检测还挺重要的。

## 坑和局限

说实话我也不确定这东西长期用会不会有问题。

第一个担心是 Cookie 鉴权。它不是走官方 API，是模拟网页端登录，Cookie 过期就得手动更新。如果你指望它真的 7x24 无人值守，得配合一个 Cookie 刷新的方案，项目本身没提供。

第二个问题是合规风险。闲鱼的用户协议大概率不允许自动化操作，虽然目前看平台没有大规模封号，但这个风险你得自己评估。项目 README 也写了"仅供学习交流"。

第三，砍价策略的精细度还不够。LLM 加规则引擎的思路是对的，但实际效果很依赖你写的提示词质量。我试了默认的提示词，有时候让价让得太痛快了，还需要自己调。

## 我的判断

我认为这类工具代表了一个趋势，AI Agent 正在从"高大上的企业级应用"渗透到最接地气的个人卖货场景。

6800 Star 不是偶然的。闲鱼有海量个人卖家，每个人都面临同样的问题，消息回不过来，砍价耗精力，追单顾不上。这是一个真实的、高频的、痛感很强的需求。

但我也想泼点冷水。

现阶段它更适合"半自动"模式，而不是完全放手。你设一个接管关键词（默认是句号），重要对话随时可以切回人工。把它当成一个帮你挡掉 80% 低质量咨询的过滤器，比当成一个全自动销售机器人更现实。

可能有些想法还不成熟，但我觉得真正的壁垒不在技术，在数据。谁能把自己几百笔成交的聊天记录喂进去做 fine-tuning，谁的 Agent 才是真正懂卖货的 Agent。现在大家都在用通用模型加提示词硬撑，上限不会太高。

## 如果你想试

项目地址在这里，Docker 部署也支持，适合扔到服务器上跑。

建议从一两个低价商品开始测，别一上来就挂贵重物品。先观察几天 LLM 的回复质量，调好提示词里的底价策略，再逐步扩大范围。

一个问题留给你，如果每个闲鱼卖家都用 AI 自动回复，买家也用 AI 自动砍价，最后会不会变成两个 AI 在互相博弈？
