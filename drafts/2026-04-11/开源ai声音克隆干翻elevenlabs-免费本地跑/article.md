ElevenLabs 每月最高收 1320 美元，一个开源模型在语音相似度上把它按在地上摩擦，英语 85.4% 对 61.3%。

这个模型叫 VoxCPM2，清华和 OpenBMB 搞的，Apache 2.0 协议，商用免费，`pip install voxcpm` 一行装完，8GB 显存就能跑。

我看到这组数字的时候愣了好几秒。

## 先说这东西能干什么

VoxCPM2 有四个核心能力，我按实用程度排序。

**声音克隆**，给它一段短音频，它能还原音色、口音、节奏、语速。不是那种"听着像"的还原，是在 Minimax-MLS 基准测试上，英语相似度 85.4%、中文 82.5%、阿拉伯语 79.1% 的还原。ElevenLabs 在同一个测试里分别是 61.3%、67.7%、70.6%。

差距不是一点半点。

**声音设计**，你用文字描述一个声音，"一位年轻女性，温柔甜美的声音"，它从零生成。不需要参考音频，不需要配音演员，不需要录音棚。20 亿参数的模型，拿 200 万小时语音数据训出来的，底气在这。

**可控克隆**，克隆声音的同时还能调情感和语速。"语速稍快，语气欢快"，搞定。这个能力在实际业务里特别有用，比如做有声书、做播客、做产品语音引导，同一个声音要适配不同场景。

**多语言**，30 种语言加 9 种中文方言（四川话、粤语、吴语都支持），自动检测语言，不需要手动打标签。

## 技术上怎么做到的

VoxCPM2 的架构挺有意思，它不走传统的 tokenizer 路线，而是用了一套"无 tokenizer 的扩散自回归"方案，在 AudioVAE V2 的潜空间里操作。整个管线四个阶段，LocEnc、TSLM、RALM、LocDiT，底座是 MiniCPM-4。

说人话就是，它先把音频压缩到一个高效的潜空间表示，然后用扩散模型在这个空间里生成语音，最后解码回 48kHz 的音频。这个思路绕开了传统 TTS 里离散 token 的信息损失问题。

在 RTX 4090 上，实时率 (RTF) 大约 0.30，用 Nano-vLLM 优化后能压到 0.13。比播放速度快好几倍。

## 但我要泼一盆冷水

坦率讲，社区里对这类 benchmark 碾压的反应已经很理性了。

有人提了一个很好的问题，benchmark 上赢了不代表实际体验就好。语音相似度是一个维度，但自然度、情感表达的细腻程度、长文本的稳定性，这些在 benchmark 里不一定能完全反映。ElevenLabs 做了这么多年商业产品，在边缘场景的鲁棒性、API 的稳定性、多端适配上的积累，不是一个 benchmark 数字能概括的。

还有人指出，本地部署虽然"免费"，但 8GB 显存的门槛对很多人来说并不低。你得有一张至少 RTX 3070 级别的显卡，这本身就是一笔投入。而且本地部署意味着你自己负责运维、更新、故障排查，这些隐性成本不能忽视。

我认为 VoxCPM2 真正的杀手锏不是"免费替代 ElevenLabs"，而是 Apache 2.0 协议带来的可能性。你可以拿它做二次开发，嵌入自己的产品，用 LoRA 在 5 到 10 分钟的音频上微调出定制声音模型。这些事情在 ElevenLabs 的商业框架里要么做不了，要么贵得离谱。

## 我的判断

我认为语音克隆这个赛道的商业模式正在被开源彻底重写。

ElevenLabs 收 1320 美元一个月不是因为技术壁垒有多高，是因为之前没有足够好的开源替代。现在有了。而且不是一个玩具级别的"差不多能用"，是在核心指标上碾压级别的开源方案。

可能有人觉得这个判断太激进了。毕竟 ElevenLabs 有完整的产品生态、有 API 服务、有企业级支持。这些没错，对于需要零运维、开箱即用的团队，ElevenLabs 仍然是合理选择。

但对于有技术能力的团队和独立开发者，花几个小时部署 VoxCPM2 然后永久免费使用，跟每个月交几百上千美元的订阅费，这个账太好算了。

说真的，我觉得 ElevenLabs 最该紧张的不是用户流失，而是定价权的丧失。当开源方案在核心指标上领先，你的定价锚点就没了。

## 想试的话

环境要求，Python 3.10 到 3.12，PyTorch 2.5.0 以上，CUDA 12.0 以上。

```
pip install voxcpm
```

一行搞定。模型权重会从 Hugging Face 或 ModelScope 自动下载。

如果你手头没有合适的 GPU，可以先在 Colab 上跑跑看，感受一下效果再决定要不要正式部署。

一个开放问题留给大家，当语音克隆的质量和成本门槛同时被开源拉平，配音行业、有声书行业、播客行业会发生什么？
