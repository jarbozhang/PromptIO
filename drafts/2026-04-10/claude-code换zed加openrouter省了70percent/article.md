有人在 HN 上晒了一张账单，Claude Code 月费 100 刀，经常还没用完就撞上限额。然后他做了一件事，把同样的 100 刀重新分配了一下，结果 70 刀变成了可以滚存的 OpenRouter 额度。

这不是省钱的故事，是花同样的钱但花得更聪明的故事。

## 撞墙的感觉

Claude Code 的订阅模式有个很反直觉的问题。你每个月付 100 刀，拿到一个固定的 token 额度窗口。用完了，要么等重置，要么加钱。没用完呢？过期作废。

这对「突发性使用」的开发者特别不友好。你可能连续三天在重构一个项目，token 烧得飞快，第四天就被限速了。然后下一周你在写文档、开会、摸鱼，那些付了钱的额度安静地过期。

博主 Ewan Jones 就是这种用法模式。他的形容很精准，"我不喜欢这种固定窗口的感觉，用了怕不够，不用又浪费。"

## 100 刀的重新分配

Ewan 的方案不是降低预算，而是把同样的 100 刀拆成三份。

Zed 编辑器订阅，10 刀/月。这是个用 Rust 写的编辑器，原生支持 Agent 面板。Ewan 的原话是，"你不用 Zed 就不知道 VSCode 和它的各种 fork 有多卡。"

Cursor 保留，20 刀/月。主要是为了试 Cursor 3.0 的新功能，算是观察仓位。

剩下的 70 刀全部充进 OpenRouter。关键来了，OpenRouter 的额度 365 天才过期。不是按月重置，是按年。所以你淡季不用的钱不会蒸发，旺季可以放开用。

这就是标题里"省了 70%"的真正含义，不是总花费少了 70%，而是原来锁死在 Claude Code 订阅里的 70 刀，现在变成了可以灵活调配的额度。

## 迁移过程没你想的那么复杂

坦率讲，我一开始以为这种迁移会很折腾。但实际操作就几步。

第一步，在 Claude Code 里输入 `/logout`。

第二步，在 shell 配置文件里加几个环境变量。

```
export OPENROUTER_API_KEY="你的key"
export ANTHROPIC_BASE_URL="https://openrouter.ai/api"
export ANTHROPIC_AUTH_TOKEN="$OPENROUTER_API_KEY"
export ANTHROPIC_API_KEY=""
```

注意最后一行，`ANTHROPIC_API_KEY` 必须显式设为空字符串，不能省略。这是个容易踩的坑。

第三步，用 `/status` 验证配置生效。

整个过程十分钟搞定。你还能通过环境变量指定默认使用的模型，比如把 subagent 模型设成 Opus 4.6，主模型用 Sonnet 4.6 省钱。

## 但有个问题没人提

OpenRouter 收 5.5% 的手续费。

70 刀的额度，实际可用大概 66 刀出头。这个成本不算高，但你得知道它在那里。Ewan 自己也提到，他关掉了 OpenRouter 的"数据用于产品改进"选项（这会损失 1% 折扣），并且开启了"仅使用零数据保留端点"。代价呢？部分模型不可用了，比如阿里的 Qwen 3.6 Plus。

HN 评论区有人提出了更激进的方案。用 LiteLLM 自建网关，连 5.5% 手续费都省了。但说实话我觉得对大多数人来说这属于过度优化，OpenRouter 的统一 API、用量追踪、多 provider 切换这些功能，5.5% 买到不亏。

## 社区里的真实数据

HN 上 289 分、194 条评论，讨论质量很高。几个数据让我印象深刻。

有人晒出自己三个月用了 44.5 亿 token，按 API 计价值 7713 美元，但只付了 300 刀订阅费。这说明 Claude Code 的订阅模式对重度用户来说其实是巨大的补贴。

另一位用户报告自己 30 天内生成了价值 1.5 万美元的 Opus token，用来构建了一整套语言编译器工具链。

反过来说，如果你是轻度到中度用户，订阅模式就是在补贴那些重度用户，你在为别人的用量买单。

还有人分享了自己的 OpenRouter 月账单，每月 550M token，花 600 刀，其中 Gemini 占 200 刀、Opus 占 400 刀。他的策略是用 Opus 做规划，用便宜模型做执行。

这个思路其实比单纯换工具更有价值。

## 我的判断

我认为 Claude Code 的订阅模式正在失去中间用户。

重度用户占了大便宜，他们不会走。轻度用户本来就用免费方案或者 Copilot。真正被挤出来的是每月花 50-150 刀、用量波动大的中间层。这批人恰恰是最有付费意愿、最愿意尝试新工具的群体。

可能有些想法还不成熟，但我观察到 HN 上已经有人说"最近几周大家都在离开 Anthropic"，也有人反映 Claude Code 最近的输出质量"像一个偷懒的初级开发者"。这些声音是不是代表趋势我不确定，但至少说明 Anthropic 在开发者工具市场的护城河没有想象中那么深。

Zed + OpenRouter 的组合不是完美方案。Zed 的扩展生态远不如 VSCode，内置的 Agent 功能也比较基础。但它解决了一个真实的痛点，让你的钱不再按月蒸发。

## 你可以今天就试

如果你想低成本试水，不需要一步到位。

先下载 Zed（免费），注册 OpenRouter 账号，充 20 刀试试。保留你现有的编辑器，两边并行跑一周，看看哪种模式更适合你的使用节奏。

真正值得思考的问题是，当 AI 编码工具的竞争从"谁的模型更强"转向"谁让开发者花钱花得更灵活"，Anthropic 准备好了吗？
