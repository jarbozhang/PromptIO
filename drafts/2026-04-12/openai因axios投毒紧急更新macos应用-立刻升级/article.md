# OpenAI 因为 axios 投毒事件紧急更新所有 macOS 应用，你需要立刻升级

4 月 11 日，OpenAI 官方发了一条推文，措辞非常克制，但如果你在 Mac 上装了任何 OpenAI 的应用，你现在就应该打开它点更新。

不是"建议更新"，是"必须更新"。

事情是这样的。axios，一个 npm 生态里下载量排名前列的 HTTP 请求库，被投毒了。这不是 OpenAI 自己的锅，是整个行业都中招了。OpenAI 在排查后确认，没有用户数据泄露，系统没有被入侵，软件也没有被篡改。但他们还是选择了最激进的应对方式，更新所有 macOS 应用的安全证书。

为什么要换证书？因为供应链攻击的恐怖之处不在于"偷了什么"，而在于"可能被伪造什么"。

## 一个库被投毒，全行业都要抖三抖

axios 这个库有多普及？几乎所有用 JavaScript 写后端或前端的项目都直接或间接依赖它。你打开任何一个 Node.js 项目的 node_modules，大概率能找到它。

供应链投毒的逻辑很简单。攻击者不需要攻破你的系统，只需要在你信任的依赖里埋一行代码。你 npm install 的那一刻，恶意代码就进来了。没有任何弹窗，没有任何提示。

OpenAI 的官方声明原文说得很明确，"out of an abundance of caution"，出于极度谨慎。他们更新安全证书的目的是防止一种场景，有人利用这次投毒事件伪造一个看起来像是 OpenAI 官方的 macOS 应用。你以为你在用 ChatGPT，其实你在给攻击者喂数据。

这不是假设。这是供应链攻击最经典的延伸路径。

## 你需要做的事，现在就做

打开你 Mac 上的 ChatGPT 应用，或者 OpenAI 的其他 macOS 应用，检查更新。如果有更新提示，立刻装。如果没有提示，去 OpenAI 官网重新下载最新版本。

OpenAI 在推文里给了两种方式，应用内更新，或者从官方链接下载。不要从任何第三方渠道下载。这句话在平时是老生常谈，在供应链投毒的背景下是救命建议。

## 这件事为什么值得多聊两句

坦率讲，"某个 npm 包被投毒"这种新闻每隔几个月就来一次。但这次不一样的地方在于，被影响的不是某个小众工具，是 OpenAI。

5432 个赞，484 次转发，339 条评论。社区的反应说明了一件事，大家对 AI 应用的供应链安全是真的紧张。

我自己也还在摸索这类事件该怎么系统性地防。但有几个事情是确定的。

第一，锁版本。你的 package-lock.json 和 yarn.lock 不是装饰品，是防火墙。每次 npm install 之前想一想，你真的审查过你要装的东西吗？

第二，关注上游安全公告。OpenAI 这次反应速度很快，从发现问题到发布声明可能就是几天的事。但如果你没有关注他们的官方渠道，你可能根本不知道这事发生了。

第三，不要假设大公司不会中招。OpenAI 也用 axios，OpenAI 也受影响。区别在于他们有团队专门做安全响应，而大多数中小团队连 dependabot 的告警邮件都没开。

## 我的判断

我认为这次事件是一个信号。AI 应用正在成为供应链攻击的高价值目标。

想想看，一个 AI 应用里流过的数据密度远超普通 SaaS，你的对话记录、你的代码片段、你的业务逻辑，全都在里面。攻击 AI 应用的投入产出比，对攻击者来说太划算了。

Karpathy 前两天在推文里说了一句话让我印象很深，最前沿的 AI 模型已经能够"find and exploit vulnerabilities in computer systems"。他说的是 AI 的能力边界在扩展，但反过来想，当 AI 能找漏洞的时候，攻击 AI 基础设施本身的门槛也在降低。

可能有些想法还不成熟，但我觉得接下来 12 个月，我们会看到更多针对 AI 工具链的供应链攻击。不是因为 AI 公司安全做得差，而是因为这个赛道里有太多值得偷的东西。
