# 交大学生用Claude Code做了个交易bot，一晚赚$1940

$1400本金，一觉醒来变$3340。

这不是什么量化基金的战报，是一个上海交大21岁学生用Claude Code写的自动交易机器人，跑了一晚上的真实结果。三笔交易，自动开仓，自动平了两笔，其中一单BTC做空从0.31进到0.79出。

我看到这条推文的时候，第一反应不是"牛逼"，而是"等等，Polymarket什么时候能这么玩了？"

然后我花了一晚上把他的架构拆了个底朝天。

## 这套系统到底在干什么

简单说，他做的是跨市场套利。

Polymarket是一个预测市场，用户可以对真实事件下注，比如"BTC下周能不能到7万"。这类盘口的定价经常出现偏差，因为参与者大多是散户，信息滞后。

他的bot同时监控Polymarket上50多个盘口的定价，然后用OpenClaw同步币安的BTC实时数据，做15分钟级别的超短线分析。一旦发现Polymarket某个BTC相关盘口的定价跟币安实际走势之间出现"裂缝"，立刻下手。

坦率讲，这个策略本身不新鲜。跨市场套利是量化交易最古老的套路之一。但新鲜的是执行方式。

## Claude Code在里面扮演什么角色

这是我最感兴趣的部分。

按他自己的说法，Claude Code"包揽了大部分苦力活"。具体来说，Claude Code负责两件事，策略生成和定价失效检测。它不只是帮忙写代码，它是策略引擎本身。

OpenClaw负责执行层，连接币安API拉数据、下单、同步状态。

整套系统0人工干预，全自动执行。他睡觉，bot干活。

我反复想了想，这里面最狠的设计不是策略，是他把Claude Code放在了"决策层"而不是"工具层"。大多数人用AI写交易代码，他用AI做交易决策。

这是一个很微妙但很重要的区别。

## 他是怎么做到2天就跑起来的

说实话我也不确定所有细节，但从他分享的信息能拼出大概轮廓。

启动资金$1400。一开始玩的是简单的跟单策略，后来觉得不过瘾，直接自己撸了一套定制化的监控和执行系统。开发过程只用了一个iPad当副屏，2天搞定。

这个速度让我想到一个对照组，Polymarket上有个叫planktonXD的玩家，年交易量大概61000笔，利润约$106000。也是套利策略，但人家是长期打磨出来的系统。

一个21岁学生用2天就逼近了类似的策略架构。

我认为这就是Claude Code这类工具真正改变的东西，不是让专业人士更快，而是让原本没资格上牌桌的人直接坐下来打牌。

## 风控设计才是重点

赚钱的故事谁都爱听，但我更关心的是，他怎么控制不亏钱。

三个关键设计。流动性异常时自动停机保护，这是最基础的。紧急清仓需要人工确认，说明他没有完全放弃人在回路里的控制权。亏损案例里提到在一波砸盘中回撤约3%，对于$1400的本金来说就是$42左右。

3%的最大回撤，对于一个全自动系统来说，算是相当克制了。

但我要泼一盆冷水。一晚上的数据什么都证明不了。三笔交易的样本量，任何统计学入门课都会告诉你这叫"不具备统计显著性"。$1940的利润可能是策略有效，也可能纯粹是运气。

可能有些想法还不成熟，但我觉得真正值得关注的不是他赚了多少钱，而是这套系统能不能持续跑三个月以上还活着。

## 我的判断

我认为这件事的意义不在交易本身。

一个大学生，$1400，2天，一个iPad，Claude Code加上OpenClaw，就能搭出一个跑在预测市场上的全自动交易系统。这件事如果放在两年前，你至少需要一个3人的量化团队、Python全栈能力、和几万美元的启动资金。

现在门槛被踩到了地板上。

但门槛低不代表赚钱容易。套利策略最大的敌人是同类竞争，当越来越多人用AI生成类似的套利bot，利润空间会被迅速压缩到零。这是一个"先到先得"的游戏，而这个窗口正在快速关闭。

反正我觉得，如果你对量化交易感兴趣，现在是最好的实验时机。不是因为一定能赚钱，而是因为试错成本从来没这么低过。$1400的本金，2天的时间，最坏的结果就是亏掉一个月的咖啡钱，但你会学到比任何课程都多的东西。

还记得开头那个$1400变$3340的故事吗？最值钱的不是那$1940，是他2天内走完了"从0到全自动交易系统"的整个过程。这个经验，比利润值钱太多了。
