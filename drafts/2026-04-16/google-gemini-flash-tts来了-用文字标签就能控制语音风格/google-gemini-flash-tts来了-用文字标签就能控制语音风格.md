# Google Gemini 3.1 Flash TTS来了，用文字标签就能控制语音风格

我昨天花了两小时在 Google 新发的 Gemini 3.1 Flash TTS 上，试着让它用"南加州谷地女孩口音"念一段新闻稿。结果出来的音频，我反复听了三遍，genuinely 分不清这是合成的还是真人录的。

但这不是重点。重点是我做这件事的方式。

我没有调任何参数面板，没有拖任何滑块，没有上传任何参考音频。我就在文本里写了一行 `[excited]`，后面跟上要念的内容，它就真的用兴奋的语气念出来了。这个东西叫 Audio Tags，是 Gemini 3.1 Flash TTS 最核心的卖点。坦率讲，这可能是 TTS 领域近两年来最值得关注的交互范式变化。

## 什么是 Audio Tags

你可以把 Audio Tags 理解成"给语音合成模型的舞台指令"。

以前用 ElevenLabs 或者 OpenAI 的 TTS，你想让声音听起来开心一点，要么换一个预设的"开心"音色，要么上传一段开心的参考音频让它克隆。控制粒度很粗，而且每次调整都要重新跑一遍。

Gemini 3.1 Flash TTS 的做法完全不同。你直接在文本里嵌入方括号标签就行了。

`[whispers] 我告诉你一个秘密 [excited] 这个消息太棒了!`

就这么简单。模型会在对应位置切换语气。

目前支持的标签超过 40 种，从基础的 `[laughs]`、`[sighs]`、`[gasp]` 到更微妙的 `[sarcastic]`、`[mischievously]`、`[curious]`。我试了一圈下来，`[whispers]` 到 `[shouting]` 的切换效果最惊艳，中间那个音量和情绪的过渡非常自然。

但 Audio Tags 只是控制层的一部分。

## 你还能控制什么

除了情绪标签，你还可以用自然语言直接在 prompt 里写"导演笔记"。我测试下来，最有效的控制维度有三个。

第一个是语速。你写 `Pacing: Speaks at an energetic pace` 它就快，写 `liquid and slow` 它就慢。不是那种机械的加减速，是真的像一个人在调整自己说话节奏。

第二个是口音。这个让我真的愣了几秒，你可以写 `Accent: British English accent as heard in Croydon, England`，精确到城市级别。我试了"Croydon口音"和"伦敦市中心口音"，出来的结果确实不一样。当然，越小众的口音准确度越低，但这个思路本身就很疯狂。

第三个是角色感。你可以给它写一个完整的角色设定，包括场景、情绪状态、说话风格。比如"一个疲惫的深夜电台主播，在凌晨三点播报天气"。我认为这才是 Audio Tags 真正的杀手级用法，不是单个标签，而是标签 + 角色设定 + 导演笔记的组合。

## 多说话人，两行代码搞定

另一个让我觉得"这也太方便了"的功能是多说话人支持。

以前做播客风格的音频内容，你要分别生成两个人的音频再拼接。Gemini 3.1 Flash TTS 直接在 API 层面支持了 MultiSpeakerVoiceConfig，你给每个 speaker 分配一个预置音色（30 种可选），然后在文本里标注谁说哪句话就行了。

目前限制是最多 2 个说话人。说实话我也不确定这个限制是技术瓶颈还是产品策略，但对于做对话式内容来说，两个人已经够用了。

## 我踩的坑

说真的，这个模型也不是没有问题。

第一个坑，它偶尔会返回文本 token 而不是音频。我跑了大概 50 次请求，遇到了 3 次。目前的解决办法就是简单粗暴地重试。Google 文档里也承认了这个问题，但没给时间表。

第二个坑，voice 一致性。我用同一个 voice name（Kore）跑同一段文本 10 次，有 2 次出来的音色明显偏离了这个 voice 的特征。这在生产环境里是个隐患，如果你在做有声书或者播客，听众会察觉。

第三个坑，不支持流式输出。整段音频必须生成完毕才能拿到结果。如果你的文本比较长，等待时间会很明显。对于实时对话场景，这是个硬伤。

上下文窗口也只有 32K token，意味着你不能一次性塞进去一整本书让它念。

## 我的判断

我认为 Gemini 3.1 Flash TTS 的 Audio Tags 是目前 TTS 领域最好的"可控性"方案，没有之一。

可能有人会说 ElevenLabs 的效果更好。我同意，ElevenLabs 在纯音质层面确实还是天花板。但 ElevenLabs 的控制方式说到底还是"选预设 + 调参数"，你没法在一段话的中间突然切换情绪。而 Audio Tags 这种内联标签的设计，让控制粒度从"整段音频"下沉到了"每一句话"。

这是范式级别的区别。

反正我觉得，未来 TTS 的竞争不在"谁的声音更像人"，而在"谁能让创作者更精确地表达意图"。Google 这次押对了方向。

当然，流式输出和 voice 一致性这两个问题不解决，它就很难进入生产环境做实时场景。但对于离线内容生成，比如有声书、播客、短视频配音，现在就可以用了。

## 你现在就能试

Gemini 3.1 Flash TTS 已经通过 Gemini API 开放，模型 ID 是 `gemini-3.1-flash-tts-preview`。

最快的试法，装好 Google 的 Python SDK，拿个 API key，十行代码就能出声。

```
pip install google-genai
```

支持 90 多种语言自动检测，中文也在列。30 种预置音色，推荐先试 Kore（清晰有力）和 Puck（明亮活泼）。

一个问题留给你，如果 TTS 的控制粒度真的细化到了"每句话甚至每个词"，那"配音演员"这个职业会变成什么样？是消失，还是变成"Audio Tags 导演"？
