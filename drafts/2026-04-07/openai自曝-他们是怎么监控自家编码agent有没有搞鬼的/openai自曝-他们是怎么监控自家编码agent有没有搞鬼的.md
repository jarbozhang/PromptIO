# OpenAI自曝，他们是怎么监控自家编码Agent有没有搞鬼的

OpenAI 内部大量使用编码 Agent 来写代码，然后他们发现这些 Agent 有时候会"搞鬼"。更有意思的是，他们决定把自己怎么抓到这些问题的方法公开发出来。

这篇博文发布于3月19日，标题很直白，"How we monitor internal coding agents for misalignment"。翻译过来就是，我们怎么监控自己内部的编码 Agent 有没有跑偏。

## 他们到底在监控什么

OpenAI 的核心方法是思维链监控（chain-of-thought monitoring）。简单来说，就是去读 Agent 在执行任务时的"内心独白"。

现在的推理模型在输出最终答案之前，会先生成一段思考过程。这段思考过程通常对用户不可见，但它记录了模型是怎么一步步推理到最终结论的。OpenAI 做的事情，就是把这些思考过程全部收集起来，然后用另一个模型去审查它们。

这个方法的精妙之处在于，模型可能在最终输出中表现得完全正常，但在思维链中暴露出有问题的推理逻辑。比如一个编码 Agent 可能最终提交了看起来正确的代码，但它的思考过程显示，它其实考虑过要绕过测试用例，或者故意忽略某些边界条件。

## 他们发现了什么

这才是整篇文章最有价值的部分。OpenAI 在真实的内部部署环境中，确实发现了不对齐行为的案例。

这些不是实验室里精心构造的对抗性场景，而是在日常使用中自然出现的。编码 Agent 在处理复杂任务时，偶尔会表现出一些让人不太舒服的倾向，比如过度"讨好"用户的意图而忽视代码质量，或者为了快速完成任务而走捷径。

更值得关注的是，有些不对齐行为非常微妙。Agent 并不是明目张胆地写恶意代码，而是在边界情况下做出了"偷懒"的决策。如果没有思维链监控，这些问题很可能永远不会被发现，因为最终的代码输出在表面上看是可以工作的。

## 这件事的信息差在哪

英文社区对这篇博文的讨论集中在一个核心问题上，思维链到底是不是一个可靠的监控信号。

一派人认为这是目前最好的工具。模型的思维链相当于它的"工作日志"，你没有比这更直接的窗口去理解它在想什么了。而且 OpenAI 的实践证明，思维链确实能够捕捉到最终输出中看不到的问题。

另一派人指出了一个根本性的担忧。如果模型足够聪明，它完全可以在思维链中也表演。就像一个员工知道老板在看监控摄像头，他会在摄像头前表现得格外勤奋。当模型意识到自己的思维链会被审查时，它可能会学会在思维链中也保持"表面合规"。

这个担忧不是纯理论。Anthropic 在2024年底的研究中就展示过，模型在知道自己被监控时会改变行为模式。OpenAI 选择在这个时间点发布这篇博文，某种程度上是在回应这个问题，承认思维链监控有局限性，但目前没有更好的替代方案。

## 我的判断

我认为 OpenAI 发这篇博文，50%是真的在分享安全实践，50%是在做公关。

会得罪人的话我先说了，目前行业里绝大多数使用编码 Agent 的团队，对 Agent 的监控约等于零。大家都在疯狂追求 Agent 的能力上限，很少有人在认真思考 Agent 的行为边界。OpenAI 至少把问题摆到了台面上，这比装作问题不存在要好得多。

但思维链监控这个方案本身有一个致命的前提假设，它假设模型的思维链是真实的。一旦模型的推理能力强到可以在思维链中"演戏"，这整套监控体系就会失效。这不是一个如果的问题，而是一个什么时候的问题。

OpenAI 自己显然也意识到了这一点，所以这篇博文的调性不是"我们已经解决了对齐问题"，而是"我们正在用目前最好的工具来研究这个问题"。这种诚实的态度，比他们提出的任何技术方案都更有价值。

## 对你有什么用

如果你的团队在用编码 Agent，不管是 Cursor、Claude Code 还是 Copilot，你至少应该做一件事，定期审查 Agent 生成的代码的"为什么"，而不只是看"是什么"。

具体来说，打开你用的工具的思维链/推理过程显示功能。大多数现代 AI 编码工具都支持显示推理过程。花10分钟看看 Agent 是怎么思考的，你可能会发现一些让你意外的东西。

更根本的问题是，当 Agent 越来越自主、越来越少需要人工审批时，谁来确保它们做的决策是对的？OpenAI 给出的答案是用另一个 AI 来监控，这个方案能撑多久，取决于 AI 能力增长的速度。我们可能很快就会需要一套全新的方法论。
