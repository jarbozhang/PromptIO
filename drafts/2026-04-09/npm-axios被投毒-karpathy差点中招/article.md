# Karpathy 差点被黑客搞了，你装的 npm 包安全吗？

axios，每周 3 亿次下载，几乎是 JavaScript 世界里用得最多的 HTTP 客户端。上周它被供应链投毒了。

Karpathy 发了条推说，他几天前装了个 Gmail/GCal 的 CLI 工具，依赖链里拉了 axios。他扫了一下系统，发现装到的版本是 1.13.5，刚好没中招。但依赖没锁版本，如果他晚几个小时装，拉到的就是被污染的 latest，直接 pwned。

差几个小时。

这件事让我后背发凉的点不是"axios 被投毒"本身，而是 Karpathy 这种级别的开发者，都是事后扫描才发现的。你我呢？

## 这次攻击到底怎么回事

攻击方式其实不新鲜，npm 供应链投毒的经典套路。攻击者拿到发布权限（可能是维护者账号泄露，也可能是社工），推了一个带恶意代码的新版本上去。因为绝大多数项目的 package.json 里写的是 `^1.x.x` 这种范围依赖，npm install 默认拉最新的兼容版本。

你装一个八竿子打不着的 CLI 小工具，它 depends on axios，axios 又刚好在那几个小时里被污染了，你的机器就被种了后门。

这个传播链条有多恐怖？3 亿周下载。每一个 `npm install` 都是一次轮盘赌。

安全团队发现得挺快，恶意版本很快就被 npm 撤了。但问题是，在那个窗口期里装过的人，根本不知道自己中了。

## 为什么包管理器的默认行为就是不安全的

坦率讲，这不是 npm 独有的问题。pip 也一样。

你用 pip install 装一个包，它默认拉最新版。你用 npm install，它默认按 semver 范围拉。两边的设计哲学都是"信任上游"，默认行为就是"给我最新的"。

这在十年前可能没什么问题。那时候 npm 上总共也没多少包，维护者你基本都认识。但现在 npm registry 有超过 200 万个包，依赖树动不动几百层深，你根本不知道你信任的到底是谁。

Karpathy 在推里提了几个个人防护手段，release-age constraints（只装发布超过一定天数的版本）、容器隔离。但他自己也说了，这些是"个人防御"，治标不治本。

他真正想说的是，包管理器的默认行为得改。

## 社区里的多种声音

这条推底下的讨论挺有意思。

一派人认为 lockfile 就是解决方案，项目里有 package-lock.json 或 yarn.lock，CI/CD 里用 `npm ci` 而不是 `npm install`，就能避免绝大多数问题。这说法对，但只对了一半，它保护的是已有项目的重复构建，保护不了你第一次 install 的那个瞬间。

另一派更激进，觉得 npm 应该默认锁死版本，不再用 `^` 和 `~` 这种范围依赖。`npm install axios` 应该写死 `"axios": "1.13.5"` 而不是 `"axios": "^1.13.5"`。这个改动听起来小，但会彻底改变 npm 生态的更新模式。

还有人提了一个更实际的方案，npm 应该有一个"quarantine period"，新发布的版本默认不作为 latest 分发，等安全扫描通过、等一两天社区没报问题了，再推为 latest。类似 Chrome 的灰度发布。

我是真的觉得第三种方案最靠谱。

## 我的判断

说实话，供应链安全这事，社区喊了很多年了，但包管理器的默认行为几乎没变过。npm、pip、cargo，都还是"信任上游+拉最新"的逻辑。

我认为这是一个系统性设计缺陷，不是用户教育能解决的。

你不能指望每个开发者都去配 release-age constraints，都去跑容器隔离，都去手动审计依赖树。能做到这些的人大概占开发者总数的 1%。剩下 99% 的人就是 `npm install` 一把梭，能跑就行。

说到底，安全不能靠自觉，得靠默认值。

谁控制默认值？npm、PyPI、crates.io 这些 registry。它们不改，喊再多"开发者要注意供应链安全"都是白搭。Karpathy 能扫描自己的系统，你公司那个刚入职三个月的前端实习生呢？

可能有些想法还不成熟，但我觉得方向很清楚，包管理器需要从"默认信任"转向"默认怀疑"。不是让开发者多干活，而是让工具链替开发者挡一层。

## 你现在能做什么

等不了 npm 改默认值的话，这几件事今天就能做。

第一，检查你的项目有没有 lockfile，有的话 CI 里用 `npm ci`，别用 `npm install`。

第二，考虑用 `npm config set save-exact true`，以后 install 的时候默认写死版本号而不是范围。

第三，全局装的包尤其危险，因为没有 lockfile 保护。如果你在本机 `npm install -g` 了什么东西，现在就去检查一下版本。

Karpathy 差几个小时就中招了。你运气有他好吗？
