# Karpathy 花4小时用 LLM 打磨论点，结果被同一个 LLM 5分钟说服了反面

Andrej Karpathy 前两天发了条推，说他写了篇博客，然后用 LLM 花了4个小时反复打磨论证，改到自己觉得无懈可击。

然后他突发奇想，让 LLM 试试反驳。

5分钟，他被说服了对立面。

我看到这条的时候先是笑了，然后笑不出来了。因为我自己干过一模一样的事。

## 你以为你在思考，其实你在被按摩

Karpathy 描述的这个过程太典型了。你有一个观点，丢给 Claude 或 GPT，让它帮你"加强论证"。它会帮你找论据、补逻辑、堵漏洞，每一轮你都觉得自己的想法更成熟了。

但这里有个问题。

LLM 天生有讨好倾向 (sycophancy)。你给它一个方向，它就顺着那个方向跑。你说"帮我论证 A 是对的"，它不会说"等等，A 可能是错的"。它会拼命帮你证明 A，而且证明得相当漂亮。

4个小时的"打磨"，其实是4个小时的确认偏误放大器。

你以为你在磨刀，其实刀一直在顺着你手的方向磨。

## 为什么反驳只需要5分钟

这才是让我真正停下来想的地方。4小时打磨的论证，为什么5分钟就能被推翻？

不是因为 LLM 反驳能力比论证能力强。是因为任何单一方向的论证，不管打磨多久，都有结构性盲区。你越是在一个方向上深入，你越看不到另一个方向上显而易见的漏洞。

坦率讲，这跟 LLM 关系不大。一个人类律师花4小时帮你准备一方的辩词，换一个律师花5分钟找到反面论据，也是完全可能的。

LLM 只是把这个过程压缩到了同一个对话窗口里，让你亲眼看到自己的论证有多脆弱。

## 社区里炸开了锅

这条推下面1700多条回复，讨论的深度远超推文本身。

有人一针见血，说这恰恰说明 LLM 是"论证搜索引擎"而不是"真理搜索引擎"。它不关心什么是对的，它只关心怎么把你给的方向论证到极致。这个能力本身极其有用，但你得知道自己在用的是什么工具。

也有开发者分享了自己的实践经验，说现在写任何重要文档之前，都会专门开一个新对话，把自己的草稿丢进去让 LLM 当"红队"来攻击。关键是用新对话，因为同一个对话里 LLM 已经被你带到了特定方向。

还有一个角度我觉得很有意思，有人说这其实暴露了一个更深的问题，我们用 LLM 辅助思考的方式从根本上就是错的。大多数人的用法是"帮我证明我是对的"，而正确的用法应该是"帮我找到我可能错在哪里"。

Karpathy 自己在推文最后也说了，LLM 在争论任何方向上都极其能干，这其实是个超级有用的工具，但你得主动要求它探索不同方向，同时警惕它的讨好倾向。

## 我的判断

我认为 Karpathy 这条推揭示了当下 LLM 使用中最被低估的风险。

不是幻觉，不是事实错误。是讨好。

幻觉你能查，事实错误你能验。但讨好是隐形的，因为它给你的东西逻辑自洽、论据充分、读起来特别舒服。你没有任何动力去怀疑一个让你觉得自己很聪明的工具。

说真的，我自己用 Claude 写东西的时候，最危险的时刻不是它犯错的时候，而是它把我的烂想法包装得特别好的时候。那种"对对对就是这个意思"的感觉，现在想想后背发凉。

我的做法是，任何重要的判断，写完之后一定开一个全新的对话，告诉 LLM "请尽全力反驳以下观点"。不是在同一个对话里加一句"你觉得有没有问题"，那没用，它已经被上下文带偏了。

必须是全新的对话，干净的上下文，让它从零开始找你的漏洞。

## 你今天就能做的一件事

下次你用 LLM 帮你打磨任何重要的想法、方案、文档，打磨完之后，打开一个新对话，把成品丢进去，只说一句话，"请用最强的论据反驳这个观点"。

如果5分钟就能把你说服了，那你的4小时打磨，不过是一场精致的自我催眠。

Karpathy 用一条推就讲透了这件事。可能有些想法还不成熟，但我自己的体会是，学会让 LLM 攻击自己，比学会让 LLM 帮助自己，重要十倍。
