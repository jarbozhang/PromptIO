这两天正写 Claude Ops 4.6 的反代源头全解析，结果 Claude 突然甩出 KYC 这张牌。消息一出，一帮人又哭丧，说中转站要凉。

凉个屁。这不是利空，这是大扫除。扫干净了，肉全他妈是我们的。

逻辑两条

第一条，官方那个 200 刀消耗池，用好了能顶 2000 刀花。但凡算账算得清的人，扭头就去绑卡开订阅。谁会傻到纯走 API？纯走 API 等于两千刀活活烧给 Anthropic 上供。这批人从官方池子里被挤出来，往哪儿涌？反代中转站。

第二条，反重力、AWS Bedrock 这帮反代渠道，Claude 把门关得越紧，他们门口队伍排得越长。官方焊死正门，外面的人想进去，只剩翻墙走我们开的侧门。这就叫官方免费替你截流，替你把人往怀里塞。

别跟着那帮半吊子博主喊利空。利空是留给只会死守官方号池、连反代都没摸过的炮灰的。

再说一嘴最近满天飞的 Persona 待认证。这活儿我以前就干，底裤什么颜色我比谁都清楚。

直接撂底：谁碰谁死。

不是一手货源、不知道内幕，听外面吹得天花乱坠，全是包装。真实内幕三刀见血：

一刀，一个认证挂一车人。
二刀，一个人炸了，全员连坐陪葬。
三刀，你永远查不出炸在哪个环节。

变量多到头皮发麻。IP、地区、挂载人数、身份本身的风控标记，现在又叠上 Persona。黑箱套黑箱，你想控制变量去抓鬼？抓个毛。

有中转站想批量收 Persona 搞共享认证，价格压到一百块顶天。多一毛都是智商税。

但只要一环崩了——母号被标记，或者某个人身份出事——上面挂的所有号全跟着死。死完你还找不到坟头哭，你根本不知道是哪个环节炸的。是人太多？是有人前科被风控？还是 IP 跟地区对不上号？

就这种地狱模式，你还死守国内 C 端维护官方号池？纯属嫌命长。维护成本、封号损耗、客服挨骂，全压你一个人脊梁骨上，利润薄得跟刀片似的。

我写那篇东西，本意是把小白踹出海当老板。但就算你出海，死磕官方接口也是一条绝路。国外用户 API 需求是大，但他们转头能接 OpenRouter，接官方直连，凭什么走你的？

反代中转站的优势就两把刀，简单到令人发指。

第一，价格屠夫。定价砍到官方十分之一，利润率照样干到百分之九十。卖十块，九块落袋。

第二，企业收割机。一旦你摸到国外企业客户的门，利润率直接飙上百分之两百。企业要的是稳定，是合规那张皮。你把反代链路包装漂亮了，翻倍报价人家照单全收。

所以别他妈盯着国内 C 端那点骨头渣了。站搭稳，反代链路调顺，扬帆出海。那边金矿还没人挖。

这两天就把 Claude Ops 4.6 的反代来源全剖开甩出来。等着抄作业。
