关于 Claude 等大模型中转站，我也来推荐几个绝对权威的信息源，因为我自己就是做中转站的，所以也能看清 X 上哪些中转站站长是真正在输出优质信息的。

@Pluvio9yte 雪踏乌云，讲了很多次中转站的运作逻辑，以及关于中转站和官方订阅的区别、价格、还有自己建立中转站的历程等分享

@cccchuizi 玩个锤子，packy 中转站站长，经常分享 Claude Code 的逆天更新如何应对，比如缓存优化方案，版本更新导致的问题如何解决等

@bozhou_ai 泊舟，dragoncode 中转站站长，在 X 上公开自己和乌云的中转站一步一步艰辛运转起来的经历分享，十分有启发意义，一步一脚印，持续更新中。

@xkajon Kai，纯纯的实力派(我最推荐关注)，古早中转站站长，将无良中转站以及中转站运作逻辑的底层逻辑在一步步讲清了，干货非常多，目前还在持续更新中。

@xaapi_ai 雪安随想，中转站站长，讲了很多中转站的运转经验，搭建中转站的过程。

为什么我作为中转站创作者还会把同行艾特出来，因为本身我的初心就不是挤兑、排挤同行
同时自媒体获客也是大家各凭本身，本身能够公开构建中转站并且还分享优质信息的中转站站长创作者，都值得推荐！

此外还有一些中转站站长没有推荐，是因为这篇是站在绝对对受众有利的角度，推荐的公开分享讲中转站的内容的创作者，有没有水平大家点进主页一看就知！
